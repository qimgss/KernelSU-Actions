# Kernel_Build

📱 **Android Kernel Build Project**

本项目用于在 **GitHub Actions** 中自动化编译 Android 设备内核。  

---

## 📌 项目说明

- ✅ 使用 GitHub Actions 云端编译
- ✅ 自动拉取指定内核仓库
- ✅ 支持自定义设备 & 内核配置
- ✅ 编译完成后自动上传产物（AnyKernel3）

---

## 🚀 使用方法

### 1️⃣ Fork本项目
- 点击右上角的`Fork`按钮
- 取消勾选`Copy the main branch only`
- 点击确定

---

### 2️⃣ 启用编译
- 点击上方的`Actions`
- 在左侧找到你需要的Workflow并点击
- 点击右侧的`Run workflow`并填好参数即可

---

## 📦 编译产物

编译完成后，可在 Actions 页面下载'AnyKernel3 ZIP'

## ❗ 注意事项
> [!IMPORTANT]
>关于工作流运行时长问题的数据参考
>
>| 内核类型 | 工作流类型 | 平均耗时范围 | 最大耗时 |
>|------------------|------------|----------------------|------------|
>| `一加` | `单次构建` | `1st: 19～35min 2nd: 9～19min` | `42min` |
>| `一加` | `矩阵构建` | `1st: 22～24min 2nd: 19～19min` | `24min` |
> `GKI` | `单次构建` | `1st: 20～36min 2nd: 12～29min` | `32min` |
> `GKI` | `矩阵构建` | `1st: 1h58min 2nd: 1h54min` | `1h56min` |
>
> >GKI内核单次构建普遍较快，但矩阵构建由于GitHub将同时运行最多jobs限制在20，从而导致低效率，请耐心等待，如果你要高效率可以考虑Github Pro或者自托管Runner
> >GKI内核单次构建的耗时从矩阵构建中提取，请以实际为准
